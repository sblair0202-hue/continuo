# MVP PRD

## MVP Objective

Prove that a field professional can verbally or manually recap a field visit and the system can convert that recap into structured, useful field intelligence.

## Core Loop

Voice/typed recap → transcript → AI extraction → user review → save to account memory → create tasks and contact updates.

## Primary User

A field-based medical device rep, therapy development specialist, clinical specialist, or territory manager.

## MVP User Stories

### Voice Recap

As a field rep, I want to verbally recap my visit so I do not have to type notes after driving between accounts.

### AI Extraction

As a field rep, I want the system to extract accounts, contacts, follow-ups, and activity updates so I can review instead of manually organizing.

### Account Memory

As a field rep, I want every account to have a chronological activity history so I can see the story of the account.

### Task Creation

As a field rep, I want follow-up tasks created from my recap so I do not forget next steps.

### Meeting Prep

As a field rep, I want a meeting brief before visiting an account so I can quickly remember what happened last time.

## MVP Screens Later

1. Today
2. New Recap
3. Review Extraction
4. Accounts
5. Account Detail
6. Contacts
7. Tasks
8. Meeting Prep

## Non-MVP

- Full Salesforce sync
- Gmail sync
- Calendar sync
- Team dashboard
- Push notifications
- Offline mode
- Real mobile audio recording
- HIPAA/PHI compliant patient tracking

## MVP Success Metrics

- User can submit a recap in under 3 minutes.
- AI output is useful without manual retyping.
- 80% or more of extracted tasks are usable after review.
- Account activity history becomes useful within 2 weeks.
- Meeting prep saves at least 10 minutes per visit.
